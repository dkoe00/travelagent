- (decision) PRD scope is a seminar prototype for a multi-agent travel planning system with at least three agents; the primary objective is evaluating and presenting OpenAI Agents SDK usage rather than launching a production travel app.
- (decision) Working mode is Fast path; target outcome is a functioning seminar demo prototype, with no planned post-seminar product continuation.
- (decision) Presentation will not include a live demo; the prototype still needs working OpenAI Agents SDK handoffs and tool calls so the team can credibly report framework experience and observations.
- (assumption) Draft assumes a terminal-based prototype with mock data, at least three specialist agents, working handoffs, working tool calls, and no required live demo during the presentation.
- (decision) All five Specialist Agents are fixed for MVP: Packing List, Places, Transportation, Budget, and Itinerary Planner; Coordinator coordinates shared trip state and final response.
- (decision) Prefer existing open/public APIs over self-built mock tools for travel data where feasible; PRD should preserve fallback boundaries so external API issues do not block seminar evaluation.
